# RustVid 运维说明

## 数据目录

默认数据目录为 `data/`，包含：

- `uploads/`：上传完成或上传中的原视频。
- `work/`：转码过程中的临时目录。
- `artifacts/`：MP4、HLS/m3u8 分片和 zip 包。
- `rustvid.sqlite3`：上传会话、任务和产物元数据。

## 清理策略

第一版默认保留原视频、历史任务和转码产物，直到管理员手动删除。删除任务时应同时清理原视频、工作目录、产物目录和数据库任务状态。

## 容量与并发

第一版使用硬编码保护：

- 总存储上限 200GB。
- 单文件上限 2GB。
- 同时转码任务上限 2 个。

达到容量保护线时，应先删除历史任务释放空间，再继续上传。

## FFmpeg

部署环境必须能从服务进程 PATH 中执行 `ffmpeg`。如果 FFmpeg 不存在或转码失败，任务会进入失败状态，用户可以重试或删除任务。
